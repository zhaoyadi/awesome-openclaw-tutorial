## 前提条件与推荐配置

本节说明运行 OpenClaw 所需的基本条件，以及在操作系统、即时通讯工具和部署方式方面的推荐选项。根据以下推荐配置，选择最适合当前使用场景的组合。

### 推荐配置

**操作系统**：

- **Mac（强烈推荐）**：原生支持最完善，可操作日历、备忘录、截图等系统功能
- Windows：完全可用，但部分系统集成功能受限
- Linux：适合开发者，配置灵活

**IM工具选择**：

- **国外用户**：推荐 **Telegram**（适配度最好，功能最完整）
- **国内用户**：推荐 **飞书**（现代化、开发友好、功能丰富）
- 备选：企业微信、钉钉、QQ

**部署方式**：

- **有Mac电脑**：推荐本地部署（体验最好，功能最全）
- **无Mac或想24小时运行**：推荐云端部署（成本低，稳定可靠）

#### Mac推荐理由

OpenClaw 在 Mac 上的体验是所有平台中最好的。这种优势源于 Mac 系统本身的设计理念与 OpenClaw 的使用场景高度契合，从系统集成的深度到用户体验的细节，Mac 平台为 OpenClaw 提供了最理想的运行环境。

在系统集成方面，Mac 提供了丰富的系统级 API，使 OpenClaw 能够深度融入操作系统。以日历管理为例，OpenClaw 不仅可以读取和创建日历事件，还支持设置提醒、添加参与者、设置重复规则等高级功能；备忘录操作除了基本的创建和搜索，还能自动分类、添加标签、插入图片和附件；提醒事项可以设置基于时间或地理位置的提醒，并与 iPhone、iPad 完全同步。这种深度集成让 OpenClaw 不再是独立的应用，而是成为 Mac 系统的有机组成部分。

文件系统的访问能力同样体现了 Mac 的优势。得益于 Mac 合理的文件系统设计，OpenClaw 可以高效地搜索、读取和整理本地文件。此外，Mac 的截图工具支持全屏截图、窗口截图、区域截图等多种模式，截图后可以直接进行标注、裁剪和分享，整个流程非常流畅。

跨设备协同能力则将 Mac 的优势进一步放大。通过 iCloud，在 Mac 上创建的备忘录、日历事件、提醒事项都会自动同步到其他 Apple 设备，在 Mac 上开始的工作可以在 iPhone 上继续完成，实现无缝衔接。

对于开发者而言，Mac 的 Unix 基础让开发环境配置更加自然，相比 Windows 需要安装 WSL2 才能获得类似体验，Mac 安装 Node.js、配置环境变量、运行命令行工具都非常直接。搭载 M 系列芯片的 Mac 运行 OpenClaw 几乎感觉不到性能损耗，稳定可靠的运行环境让用户可以专注于工作本身。

在隐私和安全方面，所有数据存储在本地，不会上传到云端，所有应用访问系统功能也需要用户明确授权，适合处理敏感信息的用户。

#### 飞书推荐理由

飞书是字节跳动推出的企业协作平台，在 OpenClaw 支持的所有国内平台中，飞书的整体体验最为出色。这一优势体现在用户体验、开发者友好度、功能完整性和服务稳定性四个维度。

在用户体验方面，飞书采用现代化设计理念，界面简洁美观，操作流畅自然，与消费级产品的质感接近，学习成本极低。飞书支持 Windows、Mac、Linux、iOS、Android、Web 等全平台，数据实时同步，在任何设备上都能无缝切换，不存在消息延迟或遗漏的问题。

在开发者友好度方面，飞书提供了完整的 API 文档，接口设计规范，错误提示清晰，并支持 WebSocket 长连接推送，消息实时性好。开发者工具和调试界面完善，问题排查方便，有活跃的社区可供交流。这些特性从工程角度保证了 OpenClaw 与飞书集成的稳定性。

在功能方面，飞书支持 Markdown 格式、代码高亮、表格、引用等富文本展示，可以在对话中直接创建和编辑文档、表格、思维导图，还支持卡片消息、按钮交互、表单收集等高级机器人能力，使 OpenClaw 的回复内容更加结构化和易读。

在稳定性和成本方面，飞书基于字节跳动的技术积累，消息推送稳定，服务可用性高，数据加密传输。个人用户使用免费版即可满足日常需求，无广告，对消息数量和文件大小也没有严格限制。综合以上优势，飞书是国内用户接入 OpenClaw 的首选平台。

#### Telegram推荐理由

Telegram 是全球最受欢迎的即时通讯工具之一，在 OpenClaw 支持的所有平台中，Telegram 的 Bot 适配度是最好的。这一结论源于 Telegram 从设计之初就将 Bot 作为核心功能，并为此提供了业界最完善的 API 支持。

在技术能力方面，Telegram 基于 MTProto 协议，消息推送速度极快，几乎没有延迟。Bot API 功能丰富且文档详细，支持内联模式（可在任意聊天中调用 Bot，无需切换对话）、自定义键盘按钮、Webhook 推送等高级特性。对单文件最高支持 2GB 的传输，消息数量也无上限，完全免费，在处理文件传输类任务时具有明显优势。

在用户体验方面，Telegram 界面简洁现代，支持全球所有主流平台，数据云端同步，全局搜索功能强大，可以快速定位历史消息和文件。在隐私安全方面，Secret Chat 模式支持端到端加密，客户端代码开源，支持消息自动销毁，无任何广告。

在全球化使用场景方面，Telegram 全球用户超过十亿，社区活跃，多语言支持完善，在全球大部分地区均可正常使用，非常适合需要与海外用户或团队协作的场景。

## 2.1 Mac本地部署

Mac 本地部署是 OpenClaw 体验最完整的安装方式，适合拥有 Mac 电脑且希望深度使用系统集成功能（日历、备忘录、文件管理等）的用户。本节将逐步说明从环境检查到启动验证的完整安装流程。

### 2.1.1 为什么选择Mac本地部署

Mac 本地部署与云端部署的根本差异在于：本地部署可以直接调用 macOS 的系统级 API，而云端部署只能通过网络接口访问有限功能。这种区别决定了 Mac 本地部署的不可替代性。

在系统集成方面，Mac 本地部署支持直接读取和创建日历事件（含提醒、参与者、重复规则）、创建和搜索备忘录（含自动分类、标签、附件）、设置提醒事项（支持时间和地理位置触发）、搜索和整理本地文件、调用系统截图工具，以及通过 iCloud 与 iPhone、iPad 自动同步。这些能力在云端部署模式下无法实现。

在性能方面，本地运行无网络延迟，响应速度以毫秒计。在隐私方面，所有对话记录和文件均存储在本地，不经过任何中间服务器。在成本方面，一次安装即可永久使用，无需支付服务器月租费用，仅需承担 AI 模型的 API 调用费用（通常每月 10～50 元）。

Mac 本地部署适合以下人群：拥有 MacBook 或 iMac 的用户、注重隐私和数据安全的用户、需要系统集成功能（日历、备忘录等）的用户、开发者和技术爱好者，以及需要处理敏感信息的专业人士。

### 2.1.3 安装步骤

#### 第一步：打开终端

按 `Command + 空格` 打开 Spotlight，输入 `Terminal` 或"终端"，按回车打开终端应用。如图2-1所示。

图2-1  通过 Spotlight 搜索打开终端

#### 第二步：安装OpenClaw

在终端中执行以下安装脚本，安装程序将自动检测系统环境、安装 Node.js（如未安装）、下载 OpenClaw 并配置环境变量。命令如下：

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

执行上述命令后，终端将显示安装进度，整个过程大约需要 2～5 分钟。安装完成后，终端会输出成功提示，此时可以进入下一步验证。

#### 第三步：验证安装

安装完成后，执行以下命令验证版本号是否正常输出。命令如下：

```bash
openclaw --version
```

如果终端显示版本号（如 `2026.2.9`），说明安装成功，可以继续进行初始化配置。如果未显示版本号或提示命令未找到，请参考 2.5 节的常见问题排查。

### 2.1.4 日常使用

完成安装后，日常使用只需管理 Gateway 服务的启动与停止。以下是常用命令。命令如下：

```bash
# 启动Gateway服务
openclaw gateway start

# 设置开机自启（推荐）
openclaw gateway enable

# 停止服务
openclaw gateway stop
```

启动服务后，打开浏览器访问 `http://127.0.0.1:18789/chat` 即可进入 Web UI 对话界面。

### 2.1.5 接入飞书

完成 Mac 本地部署之后，强烈建议将 OpenClaw 接入飞书，以获得更好的移动端使用体验。

单纯依赖 Web UI 使用 OpenClaw，只能在打开浏览器的情况下进行对话，无法在手机上随时发起请求，也无法接收 OpenClaw 主动推送的消息。接入飞书后，可以在手机、平板、电脑上通过飞书 App 随时与 OpenClaw 对话，支持语音消息、文件发送和富文本回复，OpenClaw 会以 Markdown 格式返回内容，包含代码高亮和表格展示。接入飞书后，OpenClaw 真正实现了"随时随地可用"，对于需要在移动场景中使用 AI 助手的用户来说，这一步骤值得优先完成。

具体接入步骤请参考第12章。配置完成后，即可在飞书中向 OpenClaw 发送文本、图片和文件，并收到结构化的 AI 回复。

## 2.2 云端一键部署

了解了 Mac 本地部署之后，对于没有 Mac 电脑或需要 OpenClaw 24 小时不间断运行的用户，云端部署是另一个理想选择。云端方案的最大特点是"永远在线"，无论本地电脑是否开机，OpenClaw 都在云端持续运行，随时响应请求。本节将介绍腾讯云 Lighthouse 和火山引擎两种主流云端部署方案。

注意：云端部署适合以下场景：无 Mac 电脑（Windows 或 Linux 用户）、需要 24 小时运行（定时任务、监控告警等）、多设备访问、团队协作、移动办公。

### 2.2.1 为什么选择云端部署

云端部署与 Mac 本地部署的核心差异在于运行位置不同。本地部署依赖用户的 Mac 电脑持续开机，而云端部署运行在云服务商的数据中心，具备更高的可用性和可靠性。

在便捷性方面，使用腾讯云、火山引擎等平台，点几下鼠标即可完成服务器创建，运行一条命令即可启动 OpenClaw，不需要手动安装 Node.js 或配置环境变量。在成本方面，腾讯云 Lighthouse 约 20 元/月（99 元/年），火山引擎约 9.9 元/月（58 元/年），新用户通常还有额外优惠活动。

在可访问性方面，通过飞书、企业微信、QQ 等工具，可以在手机上随时访问云端 OpenClaw，公司电脑、家里电脑和手机均可使用同一实例，选择国内云服务器则无需代理，访问速度快。在稳定性方面，云服务器由专业运维团队维护，不受本地电脑关机或断网影响，可配置自动备份，数据更安全。

### 2.2.2 方案对比

不同云平台各有侧重，可根据使用场景选择，见表2-2。

表2-2  云端部署方案对比

| 方案 | 价格 | 带宽 | 推荐场景 |
|------|------|------|----------|
| 腾讯云Lighthouse | 20元/月，99元/年 | 20M | QQ、企微用户 |
| 火山引擎 | 9.9元/月，58元/年 | 5M | 飞书用户 |
| 百度智能云 | 0.01元/月（首月） | - | 试用体验 |

腾讯云 Lighthouse 带宽更大，适合文件传输频繁或并发用户较多的场景；火山引擎价格更低，适合飞书用户日常对话使用；百度智能云首月费用极低，适合不确定是否长期使用、希望先试用的用户。

### 2.2.3 腾讯云Lighthouse部署

本节介绍使用腾讯云轻量应用服务器（Lighthouse）部署 OpenClaw 的完整流程，共分四个步骤。整个过程约需 15～20 分钟，适合没有服务器使用经验的用户。

#### 第一步：购买服务器

在搜索引擎中搜索"腾讯云轻量应用服务器"，进入官方购买页面。腾讯云经常有新用户优惠活动，建议先查看最新活动页面，再决定购买套餐。如图2-10所示。

图2-10  腾讯云 Lighthouse 控制台

在配置选择页面，按以下参数填写：

- **配置**：2核2G（满足 OpenClaw 日常运行需求）
- **带宽**：20M（支持多用户并发访问）
- **地域**：建议选择**硅谷**，访问国际 AI 模型 API 时延迟更低
- **价格**：约 20 元/月 或 99 元/年

点击"立即购买"，完成支付后等待服务器创建，通常在 1～3 分钟内完成。服务器创建完毕后，在控制台点击头像，进入"站内信"页面，记录公网 IP、用户名和登录密码，后续步骤需要用到这些信息。

#### 第二步：连接服务器

服务器就绪后，需要通过 SSH 远程登录进行后续配置。有两种连接方式可选。

**方式一：使用SSH客户端（推荐）**

下载 Termius 或 FinalShell 等 SSH 客户端，新建连接时填入以下信息：地址为服务器公网 IP，端口为 22，用户名为 `lighthouse`，密码为购买时设置的密码。连接成功后，会进入服务器的命令行终端，此时可以开始执行后续命令。

**方式二：使用网页终端**

如果不想安装额外软件，也可以在腾讯云控制台的服务器列表页面，直接点击"登录"按钮，通过浏览器打开网页终端，效果与 SSH 客户端相同，适合临时操作。

#### 第三步：配置大模型

成功登录服务器后，进入 OpenClaw 控制面板的"应用管理"标签，按以下步骤完成模型配置。如图2-11所示。

图2-11  选择 AI 模型供应商

### 2.2.4 火山引擎部署

了解了腾讯云 Lighthouse 的部署流程之后，本节介绍火山引擎的部署方案。火山引擎是字节跳动旗下的云计算平台，与飞书同属一个生态，因此对飞书用户具有额外的兼容优势，同时价格也更为低廉。

**与腾讯云的差异对比**：

火山引擎的套餐价格约为 9.9 元/月（58 元/年），低于腾讯云同规格套餐，但默认带宽仅为 5M，适合以文字对话为主、文件传输需求较少的场景。对于飞书用户而言，火山引擎的服务器与飞书消息推送服务同处字节跳动网络环境，消息延迟更低，连接更稳定。腾讯云则带宽更大，更适合需要频繁传输文件或多用户并发使用的场景。

**部署步骤**：

在搜索引擎中搜索"火山引擎云服务器"，进入官方购买页面。火山引擎针对新用户通常有折扣活动，实际价格以购买时页面展示为准。选择配置时，推荐 2 核 2G、5M 带宽规格，地域建议选择北京或上海，以便与国内飞书消息服务保持低延迟连接。

**注意事项**：

1. 购买后同样需要在控制台记录公网 IP 和登录凭据。
2. 部署流程与腾讯云 Lighthouse 基本一致，区别仅在于控制台界面布局不同，安全组配置入口名称也略有差异（火山引擎称为"安全组规则"）。
3. 火山引擎的 5M 带宽在高并发场景下可能出现速率限制，如果使用人数较多，建议升级带宽套餐或改用腾讯云方案。

如需了解更多云端部署细节（包括 Docker 部署和 Cloudflare 配置），请参考第3章。

## 2.3 国内一键安装

对于国内用户，OpenClaw 提供了专门优化的安装方案。使用国内镜像源，下载速度更快，默认配置了国产 AI 模型，适合不熟悉命令行操作、希望快速完成部署的用户。本节介绍该方案的安装步骤与验证方法。

注意：国内一键安装方案适合网络环境受限的用户，使用国内镜像源，无需代理即可完成安装。

### 2.3.1 为什么选择国内版

国内版是专门为国内用户优化的安装方案，在速度、本地化、配置简化和成本四个方面具有明显优势。

在速度方面，使用阿里云、腾讯云等国内镜像源，下载速度比直连国际源快 10 倍以上，整个安装过程通常在 1～2 分钟内完成，不会出现下载中断或连接超时的问题，也无需配置代理。

在本地化方面，所有提示和错误信息均为中文，配套中文文档和教程，有活跃的中文用户社区，默认配置符合国内用户的使用习惯。

在配置方面，一条命令完成所有配置，安装程序自动检测系统环境并智能安装依赖，提供常用配置模板开箱即用。

在成本方面，默认配置 Kimi、DeepSeek 等国产模型，价格比国际模型更低，部分模型还提供免费试用额度，可先体验后决定是否继续使用。

图2-13  国内版安装界面

## 2.5 常见问题

安装和配置过程中可能遇到以下问题。本节整理了最常见的故障现象和对应的排查方法，帮助快速定位和解决问题。

### 安装问题

**Q1：Node.js版本不符合要求**

命令如下：

```bash
# 检查当前版本
node --version

# 如果低于22，升级到22
nvm install 22
nvm use 22
```

**Q2：权限错误**

命令如下：

```bash
# macOS/Linux
sudo chown -R $USER ~/.openclaw
```

**Q3：网络连接失败**

- 检查网络连接是否正常
- 尝试配置代理后重试
- 或改用 2.3 节的国内一键安装方案

### API配置问题

**Q1：API Key无效**

- 检查是否完整复制（包含 sk- 前缀）
- 检查是否包含多余空格
- 检查账户余额是否充足

**Q2：模型不可用**

- 检查模型 ID 是否正确
- 检查 API 服务是否正常（访问对应平台的状态页面）
- 尝试切换其他模型

**Q3：Token消耗过快**

- 改用 DeepSeek 等更低价格的模型
- 优化提示词，减少不必要的上下文
- 定期清理会话历史

### Gateway问题

**Q1：Gateway无法启动**

命令如下：

```bash
# 查看日志
tail -f ~/.openclaw/logs/gateway.log

# 重启Gateway
openclaw gateway restart
```

**Q2：端口被占用**

命令如下：

```bash
# 查看端口占用情况
lsof -i :18789

# 修改Gateway端口
openclaw config set gateway.port 18790
```
