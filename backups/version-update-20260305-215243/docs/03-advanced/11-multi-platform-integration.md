# 第11章：多平台集成

本章目标是学会将OpenClaw接入飞书、企业微信、钉钉、QQ等主流平台，实现随时随地使用AI助手。掌握多平台集成后，读者可以根据自身的工作习惯和团队需求，灵活选择最合适的接入方案。

如图11-1所示，多平台集成的整体架构将OpenClaw的核心服务通过统一的Gateway层对接多个IM平台，各平台Bot独立配置、并行运行，互不影响。

## 11.1 为什么需要多平台集成

在实际使用场景中，不同用户群体依赖不同的即时通讯工具。企业员工的日常沟通主要依托飞书或企业微信，而个人用户的社交需求更多集中在QQ等平台。如果AI助手只能在一个固定入口访问，使用频率和便利性将受到明显限制。

OpenClaw通过插件化的多平台集成方案解决了这一问题。其主要优势体现在以下几个方面：

- **随时随地使用**：手机、电脑、平板均可通过已有的IM客户端访问AI助手，无需单独打开Web界面
- **团队协作**：多人可共享同一个AI助手实例，在群组中协同使用，提升团队整体效率
- **场景适配**：工作场景使用飞书，个人生活场景使用QQ，国际协作场景使用Telegram，各平台分工明确
- **统一管理**：所有平台的配置、日志和访问控制均由OpenClaw统一管理，运维成本低

理解多平台集成的价值后，下一步是根据自身情况选择合适的平台优先接入。

## 11.2 平台选择指南

选择接入平台时，需要综合考虑三个维度：一是使用者的地理位置和语言习惯，国内用户与国际用户的平台生态差异明显；二是使用场景，个人效率工具与企业办公工具对功能的侧重不同；三是平台的API成熟度，直接决定配置难度和后期维护成本。以下按国内、国外两条路径分别说明推荐选择。

### 11.2.1 国内用户推荐

对于国内用户，飞书是首选平台，企业微信和钉钉作为备选，适合有特定企业系统绑定需求的用户。

#### 飞书

飞书是当前国内对AI应用开发最为友好的IM平台，其Bot接入流程清晰，API文档质量高，且对开发者的限制较少。飞书的核心优势在于对消息格式的丰富支持：流式输出、富文本渲染、卡片消息均已实现，AI助手的回复可以获得最佳的视觉展示效果。

**适合人群**：个人效率提升用户、技术型团队、希望快速完成配置并获得完整功能体验的读者。

**核心优势**：API完善、开发友好、流式输出体验好、多Agent配置支持、基础功能完全免费。

**与OpenClaw的适配程度**：OpenClaw对飞书的支持最为完整，所有高级功能（含流式输出、卡片消息、多Agent）均可正常使用。详细配置步骤见第12章。

**适合场景**包括：个人效率提升、团队协作、项目管理以及知识管理。

#### 企业微信

企业微信的核心差异化优势是与个人微信的互通能力——企业用户可以将外部微信联系人添加为企业微信好友，AI助手可以直接面向微信用户提供服务，这一点是飞书和钉钉均不具备的。

**适合人群**：已有企业微信基础设施的公司、需要直接服务微信个人用户的客服场景、腾讯云生态用户。

**核心优势**：与微信互通、企业级安全合规、审批考勤等内置企业功能、用户基础庞大。

**与OpenClaw的适配程度**：主要功能支持完整，但不支持流式输出，长文本回复需要分段处理。详细配置步骤见第13章。

**适合场景**包括：企业内部使用、外部客户服务、工作流自动化以及已有企业微信团队的场景。

#### 钉钉

钉钉在企业办公自动化方向的积累最为深厚，其审批流程、考勤打卡、日报周报等功能与AI助手结合后，可以产生较强的场景价值。对于深度依赖钉钉进行内部管理的企业，将OpenClaw接入钉钉可以减少员工的工具切换成本。

**适合人群**：已将钉钉作为核心办公平台的企业、需要将AI能力嵌入审批或日报等流程的团队、阿里云生态用户。

**核心优势**：办公自动化功能成熟、与阿里云服务集成紧密、企业级权限管理完善、基础功能免费。

**与OpenClaw的适配程度**：核心对话功能支持完整，流式输出暂不支持，适合以问答和任务处理为主的使用场景。详细配置步骤见第13章。

**适合场景**包括：企业办公、审批流程集成、考勤管理以及已有钉钉的团队。

### 11.2.2 国际用户推荐

对于国际用户或有跨境协作需求的团队，Telegram是首选，Discord作为社区场景的备选方案。两个平台的共同优势是API开放程度高、无需企业资质认证即可创建Bot，配置门槛相对较低。

#### Telegram

Telegram在全球范围内拥有庞大的用户基础，其Bot API设计成熟，文档完善，对开发者极为友好。Telegram Bot支持流式输出，消息格式支持Markdown，AI回复的可读性好。此外，Telegram对隐私保护较为重视，端到端加密机制适合对数据安全有较高要求的场景。

**适合人群**：海外或跨境团队用户、对隐私保护有要求的个人用户、国际开源社区运营者。

**核心优势**：全球覆盖、Bot API成熟、支持流式输出、隐私保护机制完善、完全免费。

**与OpenClaw的适配程度**：适配完整，包含流式输出支持，是国际用户体验最接近飞书的平台。Telegram的基础配置在OpenClaw的通用配置向导中已内置，参照本章11.4节的通用配置流程即可完成接入。

**适合场景**包括：个人使用、国际团队协作、隐私要求较高的场景以及社区运营。

#### Discord

Discord起源于游戏社区，但近年来在技术开发者社区和创意领域的使用也逐渐普及。其独特优势在于支持语音频道，以及服务器（Server）模式下细粒度的频道权限管理，适合需要对不同话题或功能进行物理隔离的大型社区。

**适合人群**：游戏玩家社区、技术开发者社区、需要语音与文字混合交流的场景、已建立Discord服务器的社区运营者。

**核心优势**：语音频道支持、频道权限管理灵活、Bot生态成熟、社区互动氛围活跃。

**与OpenClaw的适配程度**：核心对话功能适配良好，支持富文本回复，频道Bot权限配置需要额外注意。详细配置步骤见第13章。

**适合场景**包括：游戏社区、技术社区、语音交流以及面向国际用户的社区运营。

## 11.4 通用配置流程

无论选择哪个平台，OpenClaw的接入流程在结构上是一致的，均分为四个步骤：创建应用或机器人、配置OpenClaw、测试验证、配置访问控制。以下逐步说明各阶段的操作要点。

### 步骤1：创建应用或机器人

各平台均需要在其开发者中心完成应用注册并获取凭证。具体路径因平台而异，但核心操作相同：

1. **注册开发者账号**：访问对应平台的开发者中心，注册账号并完成身份认证（部分平台需要企业资质）
2. **创建应用**：在应用管理界面新建应用，填写应用名称、描述等基本信息
3. **获取凭证**：记录 App ID、App Secret、Bot Token 等关键凭证，后续配置时需要使用

飞书和企业微信的具体创建步骤分别见第12章和第13章，两章均包含截图指引。

### 步骤2：配置OpenClaw

获取凭证后，需要在OpenClaw中安装对应的平台插件并录入凭证信息。

安装对应插件，命令如下：

```bash
# 飞书
openclaw plugins install @openclaw/feishu

# 企业微信
openclaw plugins install @openclaw/wework

# 钉钉
openclaw plugins install @openclaw/dingtalk
```

完成插件安装后，录入凭证信息。可使用配置向导（推荐新手使用），也可手动指定配置项，命令如下：

```bash
# 方式1：使用配置向导（交互式引导）
openclaw onboard

# 方式2：手动配置指定字段
openclaw config set channels.feishu.appId cli_xxx
openclaw config set channels.feishu.appSecret xxx
```

完成凭证配置后，重启Gateway使配置生效，命令如下：

```bash
# 重启Gateway
openclaw gateway restart
```

### 步骤3：测试验证

Gateway重启完成后，在对应的IM平台中找到刚创建的机器人，发送一条测试消息（如"你好"），观察是否收到AI回复。

若需要查看详细的消息收发日志，执行以下命令：

```bash
# 实时查看Gateway日志
tail -f ~/.openclaw/logs/gateway.log
```

验证以下三点：回复是否按时到达、回复内容是否来自AI、多轮对话是否连续。如有异常，参照本章11.5节的常见问题进行排查。

### 步骤4：访问控制

正式使用前，建议配置访问控制，防止未授权用户调用AI助手，避免产生不必要的API费用。

配置用户白名单，命令如下：

```bash
# 添加允许使用的用户ID
openclaw config set channels.feishu.allowlist '["user_id_1","user_id_2"]'
```

访问控制还包括以下方面：设置管理员账号以管理Bot行为、按功能维度设置权限（如哪些用户可以调用知识库功能）、对群组使用场景配置群组级白名单。权限配置的详细说明见各平台专属章节。

## 11.5 常见问题

本节汇总读者在多平台集成过程中最常遇到的问题及对应处理方法。

**应该选择哪个平台？**

根据使用场景做出选择：个人使用首选飞书，功能最完整且体验最佳；企业使用推荐企业微信，可与微信互通；有钉钉基础的团队直接接入钉钉；国外用户选择Telegram，全球覆盖且API成熟。

**可以同时配置多个平台吗？**

可以。OpenClaw支持同时接入多个平台，各平台的插件独立运行，互不干扰。例如工作用飞书、个人生活用QQ、社区运营用Discord，三个平台可以并行使用同一个AI助手后端，通过统一的Gateway进行路由。

**配置后无法收到消息怎么办？**

按以下顺序排查：第一步检查Gateway运行状态（`openclaw gateway status`）；第二步查看实时日志（`tail -f ~/.openclaw/logs/gateway.log`）；第三步验证录入的凭证是否正确，注意区分 App ID 与 App Secret；第四步检查服务器网络是否能正常访问对应平台的API端点；第五步参阅第12章（飞书）或第13章（企业微信、钉钉、QQ）中对应平台的故障排查章节。

**如何配置多个Agent？**

多Agent配置需要在平台侧创建多个Bot应用，并在OpenClaw中分别绑定不同的Agent配置。具体操作步骤见第12章的飞书多Agent配置说明，以及第13章的企业微信多Agent配置说明，两章中的配置思路可以相互参考。

**配置成本是多少？**

大部分平台的基础功能均免费。飞书、钉钉、Discord的高级功能可选付费；企业微信、QQ、Telegram完全免费。对于个人用户和中小团队，免费功能已足够满足日常AI助手使用需求。

## 本章小结

本章系统介绍了OpenClaw多平台集成的整体思路与操作框架。核心内容包括四个部分：

第一，平台选择指南。国内用户优先选择飞书，有企业系统绑定需求的团队选择企业微信或钉钉，国际用户选择Telegram，社区场景选择Discord。选择时重点关注流式输出支持、API成熟度以及目标用户的活跃平台。

第二，平台功能对比。飞书和Telegram在功能完整性与开发友好度上综合评分最高；企业微信和钉钉的优势在于与企业办公流程的深度集成，而非功能全面性。

第三，通用配置流程。所有平台均遵循"创建应用获取凭证—安装插件录入凭证—重启Gateway—测试验证—配置访问控制"的标准流程，熟悉一个平台的配置后，其他平台可快速类比完成。

第四，选择建议。建议读者从一个平台开始，完整走通配置流程并验证功能后，再根据需要扩展到其他平台。首次配置推荐选择飞书，原因是其API文档最清晰、排错成本最低，且OpenClaw对其支持最为完整。

各平台的详细配置步骤分别见第12章（飞书）和第13章（企业微信、钉钉、QQ）。
