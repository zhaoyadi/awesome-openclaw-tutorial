# 第13章：企业微信、钉钉与QQ配置

本章介绍如何将 OpenClaw 接入企业微信、钉钉和 QQ 三个平台。这三个平台在国内拥有庞大的用户基础，分别面向不同的使用场景：企业微信以与微信生态打通为核心优势，适合需要对外服务的企业；钉钉深耕办公自动化，适合审批流程和考勤管理需求较强的企业；QQ 则以个人社交为主，适合兴趣社区和个人开发者。读者可根据实际需求选择其中一个或多个平台接入。

安装与基础配置请参阅第2章，多平台接入的统一架构说明请参阅第11章，飞书的接入方式已在第12章详细介绍。

## 13.1 企业微信Bot配置

### 企业微信平台概述

企业微信是腾讯面向企业用户推出的办公协作工具，其最显著的特点是与微信生态深度打通：企业可以向外部微信用户发送消息，用户无需下载企业微信客户端也能接收通知。这一特性使企业微信在客户服务、对外通知推送等场景中具有不可替代的优势。

在 OpenClaw 的多平台接入方案中，企业微信频道适用于以下典型场景：团队内部的 AI 助手服务（成员通过企业微信与 Bot 对话）、面向微信用户的客户服务机器人，以及与企业工作流系统联动的自动化通知。企业微信的基础功能完全免费，且没有用户数量限制，部署成本较低。

### 创建企业微信应用

#### 步骤1：注册企业微信账号

如果所在企业尚未开通企业微信，需要先完成注册。访问企业微信开发者后台（在搜索引擎中搜索"企业微信开放平台"即可找到入口），点击"立即注册"，选择"企业注册"并填写企业名称、行业类型、企业规模和管理员信息，验证手机号后完成注册。已有企业微信账号的团队可直接跳过此步骤。

#### 步骤2：创建自建应用

注册完成后，使用管理员账号登录企业微信管理后台，进入"应用管理"页面，点击"创建应用"。填写以下信息：

- **应用名称**：填写 OpenClaw Bot 或自定义名称
- **应用介绍**：简要说明用途，例如"AI 对话助手"
- **应用Logo**：可上传图标，也可跳过

点击"创建应用"后，系统会生成应用并跳转至应用详情页。

#### 步骤3：获取凭证信息

应用详情页包含接入所需的三项核心凭证。请逐项记录，后续配置 OpenClaw 时需要用到：

- **Corp ID**（企业ID）：位于管理后台首页的"企业信息"模块
- **Agent ID**（应用ID）：位于应用详情页顶部
- **Secret**（应用密钥）：位于应用详情页的"Secret"字段，首次查看时需要手机验证

需要注意的是，Secret 在创建后只会完整显示一次，必须立即复制并保存至安全位置。若不慎丢失，只能重新生成，旧 Secret 将立即失效。

#### 步骤4：配置应用权限与可见范围

凭证记录完毕后，还需要为应用配置消息收发权限。在应用详情页点击"功能"选项卡，开启以下权限：

- 接收消息（允许用户主动向 Bot 发送消息）
- 发送消息（允许 Bot 向用户推送回复）
- 通讯录管理（若需要识别用户身份，可选择开启）

同时，在"可见范围"配置中选择"全部成员"，或指定特定部门和成员。可见范围决定了哪些员工能在工作台中看到并使用该应用。完成上述配置后，企业微信开发者后台的操作即告完成。

### 配置OpenClaw接入企业微信

凭证准备就绪后，可通过以下三种方式完成 OpenClaw 的配置。推荐首次接入的用户使用配置向导，熟悉流程后可改用手动配置或命令行方式。

#### 方式一：使用配置向导

运行以下命令启动交互式配置向导：

```bash
openclaw onboard
```

在向导中依次选择 Configure Channels、WeWork，然后按提示输入 Corp ID、Agent ID 和 Secret，向导完成后配置自动写入。

#### 方式二：手动编辑配置文件

直接编辑配置文件 `~/.openclaw/openclaw.json`，在 `channels` 字段下添加企业微信配置节：

```json
// ~/.openclaw/openclaw.json（企业微信配置节示例）
{
  "channels": {
    "wework": {
      "enabled": true,
      "corpId": "ww_xxxxxxxxxx",
      "agentId": "1000001",
      "secret": "your_secret_here"
    }
  }
}
```

其中 `corpId`、`agentId` 和 `secret` 替换为步骤3中记录的实际值。

#### 方式三：使用命令行逐项写入

如果不希望直接编辑 JSON 文件，也可以使用 `openclaw config set` 命令逐项写入：

```bash
# 第13章/openclaw-config-wework.sh
# 配置 Corp ID
openclaw config set channels.wework.corpId ww_xxxxxxxxxx

# 配置 Agent ID
openclaw config set channels.wework.agentId 1000001

# 配置 Secret
openclaw config set channels.wework.secret your_secret_here

# 启用企业微信频道
openclaw config set channels.wework.enabled true
```

三条写入命令执行完毕后，最后一条命令将频道置为启用状态，配置即生效。

### 测试与验证

配置写入后，需要重启 Gateway 使新配置生效，然后通过实际发送消息来验证接入是否成功。

**重启 Gateway 并检查状态**

执行以下命令重启 Gateway 并确认运行状态：

```bash
# 重启 Gateway
openclaw gateway restart

# 检查运行状态
openclaw gateway status
```

状态输出中若显示企业微信频道为 `running`，说明 Gateway 已成功加载配置。

**在企业微信中发送测试消息**

打开企业微信客户端，点击底部导航栏的"工作台"，找到刚创建的应用（OpenClaw Bot），点击进入对话界面，发送以下测试消息：

```
你好，能听到我说话吗？
```

Bot 正常响应即表示配置成功。若 Bot 在30秒内未回复，可参阅下方常见问题进行排查。

### 常见问题（企业微信）

**Q1：发送消息后 Bot 无响应**

首先确认 Gateway 是否正在运行（`openclaw gateway status`），其次逐项核对 Corp ID、Agent ID、Secret 是否与开发者后台完全一致（注意区分大小写和首尾空格）。此外还需确认应用的可见范围包含当前测试账号，以及"接收消息"和"发送消息"权限均已开启。

**Q2：如何限制只有特定用户可以使用 Bot**

通过用户白名单可以控制访问权限。执行以下命令，将允许使用的用户 ID 写入配置：

```bash
# 配置用户白名单（用户 ID 来自企业微信后台）
openclaw config set channels.wework.allowlist '["user1","user2"]'
```

**Q3：如何查看运行日志进行故障排查**

企业微信频道的运行日志保存在以下路径，可实时跟踪输出：

```bash
# 实时查看企业微信频道日志
tail -f ~/.openclaw/logs/wework.log
```

## 13.2 钉钉Bot配置

与企业微信专注于微信生态打通不同，钉钉的核心竞争力在于办公自动化。作为阿里巴巴旗下的企业协作平台，钉钉与阿里云服务深度集成，其审批流程、智能考勤、DING 消息等功能在大型企业中有广泛应用。对于已经将内部流程迁移至钉钉的企业而言，在钉钉中接入 AI Bot 可以直接复用现有的员工账号体系和权限管理机制，无需额外建立用户管理系统。

### 钉钉平台概述

钉钉在 OpenClaw 接入体系中适用于以下场景：在审批流程中引入 AI 辅助决策（如自动提取合同关键信息供审批人参考）、在企业内部 IM 中提供智能问答服务，以及与考勤、日报等钉钉原生功能联动的自动化场景。钉钉的基础功能对企业免费开放，开放平台的文档体系相对完善，适合有一定开发基础的技术团队接入。

### 创建钉钉应用

#### 步骤1：登录钉钉开放平台

访问钉钉开放平台开发者后台（在搜索引擎中搜索"钉钉开放平台"即可找到入口），使用钉钉账号登录。若是首次访问开发者后台，需要完成开发者认证，填写真实姓名和联系方式即可完成。

#### 步骤2：创建企业内部应用

登录成功后，进入"开发者后台"主页，点击"创建应用"，在弹出的应用类型选择中选择"企业内部应用"。填写以下信息后点击"确认创建"：

- **应用名称**：填写 OpenClaw Bot 或自定义名称
- **应用描述**：简要说明用途，例如"企业内部 AI 对话助手"
- **应用图标**：可上传图标，也可跳过

企业内部应用只对本企业员工可见，与"第三方企业应用"（面向所有钉钉企业的应用市场）不同，此处应选择企业内部应用。

#### 步骤3：获取凭证信息

应用创建完成后，进入应用详情页的"基础信息"页签，记录以下两项凭证：

- **App Key**：应用唯一标识，相当于企业微信的 Agent ID
- **App Secret**：应用密钥，用于接口鉴权

App Secret 与企业微信的 Secret 一样，在首次创建后需要及时复制保存。若后续需要重置，旧密钥将立即失效，已接入的服务需要同步更新配置。

#### 步骤4：配置应用权限

进入应用详情页的"权限管理"页签，开启以下权限以保证消息收发功能正常运行：

- 企业内部通讯录只读权限（用于识别用户身份）
- 消息通知权限（允许 Bot 主动推送消息）
- 机器人消息权限（允许用户向 Bot 发送消息）

在"发布管理"中，将应用的可见范围设置为"全部员工"或指定部门。权限配置完成后，应用需要经过发布流程才能对员工生效，按照页面提示完成发布即可。

### 配置OpenClaw接入钉钉

钉钉频道的配置方式与企业微信类似，同样提供三种途径。完成企业微信接入的读者会发现钉钉的操作流程基本一致，主要区别在于凭证字段的名称不同。

#### 方式一：使用配置向导

运行以下命令启动交互式配置向导：

```bash
openclaw onboard
```

在向导中依次选择 Configure Channels、DingTalk，然后按提示输入 App Key 和 App Secret，向导完成后配置自动写入。

#### 方式二：手动编辑配置文件

直接编辑配置文件 `~/.openclaw/openclaw.json`，在 `channels` 字段下添加钉钉配置节：

```json
// ~/.openclaw/openclaw.json（钉钉配置节示例）
{
  "channels": {
    "dingtalk": {
      "enabled": true,
      "appKey": "ding_xxxxxxxxxx",
      "appSecret": "your_secret_here"
    }
  }
}
```

#### 方式三：使用命令行逐项写入

```bash
# 第13章/openclaw-config-dingtalk.sh
# 配置 App Key
openclaw config set channels.dingtalk.appKey ding_xxxxxxxxxx

# 配置 App Secret
openclaw config set channels.dingtalk.appSecret your_secret_here

# 启用钉钉频道
openclaw config set channels.dingtalk.enabled true
```

三条命令依次执行完毕后，最后一条命令将频道置为启用状态，Gateway 重启后即可生效。

### 测试与验证

**重启 Gateway 并检查状态**

执行以下命令重启 Gateway 并确认钉钉频道的运行状态：

```bash

# 检查运行状态

```

状态输出中钉钉频道显示 `running` 即表示配置已加载。

**在钉钉中发送测试消息**

打开钉钉客户端，点击底部导航栏的"工作台"，找到刚发布的应用（OpenClaw Bot），点击进入对话界面，发送以下测试消息：

```
你好，能听到我说话吗？
```

Bot 正常响应即表示配置成功。

### 常见问题（钉钉）

**Q1：发送消息后 Bot 无响应**

依次检查以下几项：确认 Gateway 正在运行，确认 App Key 和 App Secret 与开发者后台完全一致，确认机器人消息权限已开启，确认应用已发布且当前测试账号在可见范围内。钉钉的"应用发布"步骤是很多开发者遗漏的环节——即使权限配置正确，未发布的应用也无法在工作台中被员工看到。

**Q2：如何限制只有特定用户可以使用 Bot**

```bash

openclaw config set channels.dingtalk.allowlist '["user1","user2"]'
```

**Q3：如何查看运行日志进行故障排查**

```bash

tail -f ~/.openclaw/logs/dingtalk.log
```

## 13.3 QQ Bot配置

企业微信和钉钉主要面向企业办公场景，而 QQ 的定位截然不同。QQ 是国内历史最悠久、用户基数最大的即时通讯工具之一，其用户群体以年轻人、学生和游戏玩家为主，在兴趣社区、游戏辅助和个人开发者场景中具有天然优势。对于不需要企业认证、希望快速面向个人用户提供 AI 服务的开发者而言，QQ 是门槛较低的接入选择。

### QQ平台概述

QQ Bot 通过 QQ 开放平台（即 Q.QQ.COM 开放平台，以下简称 QQ 开放平台）进行管理。与企业微信和钉钉以企业为主体不同，QQ 开放平台同时支持个人开发者和企业开发者注册，审核流程相对灵活。接入后，Bot 可以作为 QQ 好友与用户进行私聊，也可以被邀请进入 QQ 群提供群内服务。

在 OpenClaw 的适用场景中，QQ 频道适合：面向学生群体的学习辅助 Bot、游戏社区的玩法问答 Bot、以及个人开发者在朋友圈子中测试和分发自己的 AI 应用。

### 创建QQ机器人

#### 步骤1：注册QQ开放平台账号

访问 QQ 开放平台（在搜索引擎中搜索"QQ 机器人开放平台"即可找到入口），使用 QQ 账号扫码登录。首次登录需要完成开发者认证，填写开发者信息后提交审核，通常在1个工作日内完成。

#### 步骤2：创建机器人

开发者认证通过后，进入"机器人管理"页面，点击"创建机器人"。填写以下信息：

- **机器人名称**：填写 OpenClaw Bot 或自定义名称
- **机器人简介**：简要说明用途
- **机器人头像**：可上传图标，也可跳过

点击"创建"后，系统进行信息审核，审核通过后机器人即可使用。

#### 步骤3：获取凭证信息

机器人审核通过后，进入机器人详情页，记录以下两项凭证：

- **App ID**：机器人的唯一标识符
- **Token**：接口鉴权令牌

Token 在首次创建后需要及时复制保存，重置后旧 Token 立即失效。建议将凭证保存在密码管理工具或安全的配置文件中，不要以明文形式保存在代码仓库里。

#### 步骤4：配置机器人权限

进入机器人详情页的"权限管理"模块，开启以下权限：

- 接收私聊消息（允许用户以好友身份向 Bot 发送消息）
- 发送消息（允许 Bot 回复用户）
- 群消息相关权限（若需要群内服务，可选择开启）

部分权限需要额外的业务说明才能通过审核，建议在申请时如实填写使用场景。

### 配置OpenClaw接入QQ

QQ 频道的凭证字段为 App ID 和 Token，配置方式与前两个平台一致，三种途径均可选用。

#### 方式一：使用配置向导

运行以下命令启动交互式配置向导：

```bash
openclaw onboard
```

在向导中依次选择 Configure Channels、QQ，然后按提示输入 App ID 和 Token，向导完成后配置自动写入。

#### 方式二：手动编辑配置文件

直接编辑配置文件 `~/.openclaw/openclaw.json`，在 `channels` 字段下添加 QQ 配置节：

```json
// ~/.openclaw/openclaw.json（QQ 配置节示例）
{
  "channels": {
    "qq": {
      "enabled": true,
      "appId": "your_app_id",
      "token": "your_token_here"
    }
  }
}
```

#### 方式三：使用命令行逐项写入

```bash
# 第13章/openclaw-config-qq.sh
# 配置 App ID
openclaw config set channels.qq.appId your_app_id

# 配置 Token
openclaw config set channels.qq.token your_token_here

# 启用 QQ 频道
openclaw config set channels.qq.enabled true
```

三条命令依次执行完毕后，最后一条命令将频道置为启用状态。

### 测试与验证

**重启 Gateway 并检查状态**

执行以下命令重启 Gateway 并确认 QQ 频道的运行状态：

```bash
# 重启 Gateway
openclaw gateway restart

# 检查运行状态
openclaw gateway status
```

状态输出中 QQ 频道显示 `running` 即表示配置已加载。

**在QQ中发送测试消息**

打开 QQ 客户端，在搜索框中搜索机器人的 QQ 号或名称，添加为好友后进入对话界面，发送以下测试消息：

```
你好，能听到我说话吗？
```

Bot 正常响应即表示配置成功。若需要测试群内服务，可将机器人邀请进入测试群后发送带有 `@机器人` 前缀的消息。

### 常见问题（QQ）

**Q1：发送消息后 Bot 无响应**

依次检查以下几项：确认 Gateway 正在运行，确认 App ID 和 Token 与开发者后台完全一致，确认私聊消息接收权限已开启，以及确认机器人账号的好友申请已通过（QQ Bot 默认需要通过好友验证才能接收私聊消息，可在开发者后台的消息设置中调整验证方式）。

**Q2：如何限制只有特定用户可以使用 Bot**

```bash
# 配置用户白名单（用户 ID 为对方的 QQ 号）
openclaw config set channels.qq.allowlist '["user1","user2"]'
```

**Q3：如何查看运行日志进行故障排查**

```bash
# 实时查看 QQ 频道日志
tail -f ~/.openclaw/logs/qq.log
```

## 13.4 三平台对比与选择建议

完成三个平台的配置介绍后，有必要对它们的差异做一次系统性梳理，帮助读者在实际项目中做出合理的平台选择。三个平台在技术上都能与 OpenClaw 稳定集成，差异主要体现在用户覆盖范围、办公功能深度和开发者体验等维度。

表13-1 三平台主要特性对比

| 特性 | 企业微信 | 钉钉 | QQ |
|------|----------|------|----|
| 与微信生态互通 | 支持 | 不支持 | 不支持 |
| 办公自动化能力 | 良好 | 优秀 | 不适用 |
| 审批流程集成 | 良好 | 优秀 | 不支持 |
| 国内用户基数 | 非常大 | 较大 | 非常大 |
| 开发者文档完善度 | 良好 | 良好 | 一般 |
| 企业认证要求 | 需要企业认证 | 需要企业认证 | 个人开发者可用 |
| 基础功能费用 | 免费 | 免费 | 免费 |
| 主要适用场景 | 企业办公、客户服务 | 企业办公、审批考勤 | 个人社交、兴趣社区 |

### 选择建议

面向对外客户服务或希望覆盖微信用户的企业，优先选择企业微信。企业微信与个人微信的互通能力是其他平台无法复制的差异化优势，尤其适合需要向非员工用户推送通知或提供客服的场景。同时，企业微信的通讯录管理功能可以与现有人事系统对接，简化用户身份管理。

内部流程已深度依赖钉钉的企业，应选择钉钉接入。钉钉的审批流程和 DING 强提醒消息是其核心优势，若 AI Bot 需要参与审批辅助、自动填写日报或集成考勤数据等场景，钉钉能够提供最低摩擦的集成路径。此外，与阿里云服务的天然集成也使得钉钉在已使用阿里云基础设施的企业中具有额外优势。

个人开发者或面向年轻用户群体的项目，选择 QQ 门槛最低。QQ 不要求企业资质认证，个人开发者即可完成注册和接入，开发和测试周期短。对于学习型项目、兴趣社区 Bot 或者希望快速验证产品想法的场景，QQ 是最灵活的起点。

三个平台也可以同时接入。OpenClaw 的多频道架构（详见第11章）支持 Gateway 同时监听多个平台，Bot 的核心逻辑和 AI 模型配置统一管理，无需为每个平台分别部署服务。这意味着已经接入企业微信的团队，可以在几分钟内完成钉钉或 QQ 的追加配置。

## 本章小结

本章系统介绍了企业微信、钉钉和 QQ 三个平台的接入方法。三个平台的配置流程遵循相同的模式：在各平台开发者后台创建应用并获取凭证，然后通过 OpenClaw 的三种配置方式（配置向导、手动编辑、命令行写入）将凭证写入 Gateway 配置，最后重启 Gateway 并通过实际消息验证接入效果。

企业微信适合需要微信生态互通的企业，钉钉适合深度依赖办公自动化的企业，QQ 适合个人开发者和面向年轻用户的项目。三者并非互斥选择，OpenClaw 的多平台架构支持同时接入，统一管理。

关于多平台统一管理的进阶配置，请参阅第11章。下一章（第14章）将介绍通过 API 方式将 OpenClaw 集成至自定义应用程序的方法，适合需要将 AI 能力嵌入现有业务系统的场景。
