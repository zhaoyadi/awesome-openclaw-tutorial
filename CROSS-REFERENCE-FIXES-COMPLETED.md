## 📋 交叉引用原则（已明确）

### 应该避免的交叉引用

为了避免"挖坑"，以下类型的交叉引用应该尽量避免：

❌ **避免使用**：
1. "详见第X章"（如果第X章没有对应内容，读者会困惑）
2. "参见第X章"（同上）
3. "如第X章所述"（同上）
4. 括号内的章节引用，如"（见第8章）"

### 推荐的引用方式

✅ **推荐使用**：
1. **章节导航链接**（章节开头和结尾）
   ```markdown
   - [← 上一章](链接)
   - [→ 下一章](链接)
   ```

2. **"下一步"部分的链接**（章节结尾）
   ```markdown
   ## 下一步
   - 📖 进阶部署：[第3章](链接)
   - 🚀 快速上手：[第5章](链接)
   ```

3. **直接链接**（不强调章节号）
   ```markdown
   详细配置请参考[飞书Bot配置](链接)
   ```

### 为什么要避免交叉引用？

1. **内容可能变化**：章节内容可能调整，引用的内容可能不存在
2. **读者体验差**：读者找不到引用的内容会很困惑
3. **维护成本高**：每次调整内容都要检查所有引用
4. **容易出错**：章节重新编号后容易遗漏更新

## 💡 建议

### 1. 减少交叉引用

**原则**：每章内容应该相对独立，尽量不依赖其他章节

**示例**：
```markdown
# 不推荐
详细配置方法请参见第12章。

# 推荐
详细配置方法请参考[飞书Bot配置](链接)。
```

### 2. 使用描述性链接

**原则**：链接文字应该描述目标内容，而不是章节号

**示例**：
```markdown
# 不推荐
参见第12章

# 推荐
参考[飞书Bot配置](链接)
```

### 3. 章节导航使用相对路径

**原则**：使用相对路径，便于文件移动

**示例**：
```markdown
# 推荐
[← 上一章：快速安装](02-installation.md)
[→ 下一章：进阶部署](03-advanced-deployment.md)
```
