# 交叉引用和URL清理计划

## 📋 任务目标

1. **交叉引用准确性**：确保所有章节间的引用链接准确无误
2. **URL清理**：移除书中的具体网址，改为描述性文字
3. **价格时间限定**：所有涉及价格和数字的地方加上时间限定（如"2026年初"）

---

## 🔍 问题分析

### 问题1：交叉引用不准确

**风险**：
- 章节重新编号后，内部链接可能失效
- "参见第X章"的引用可能指向错误章节
- 导航链接（上一章/下一章）可能不正确

**需要检查的位置**：
- 每章开头的"本章导航"
- 每章结尾的"下一步"链接
- 正文中的"参见第X章"引用
- README.md中的章节链接

### 问题2：网址和价格问题

**风险**：
- 网址内容可能变化（价格、政策等）
- 价格信息可能过时
- 缺少时间限定，读者不知道信息时效性

**需要处理的内容**：
- 云服务商价格（腾讯云、阿里云、火山引擎等）
- API服务价格（DeepSeek、Kimi、Claude等）
- 具体网址链接
- 数字相关的描述

#### 2.2 URL处理原则

**需要移除的URL**：
- 云服务商购买页面
- API服务商定价页面
- 第三方服务商链接

**可以保留的URL**：
- OpenClaw官方网站
- OpenClaw官方文档
- GitHub开源项目
- 技术标准文档

**替换方式**：
```markdown
# 原文
访问腾讯云官网：https://cloud.tencent.com/

# 修改为
访问腾讯云官网（搜索"腾讯云"即可找到）
```
