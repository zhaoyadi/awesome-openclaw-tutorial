## 详细问题清单

### 一、标题格式问题（3个）

#### 1. ⭐ 标题中有注释性内容

**问题位置**：
- `## 2.1 Mac本地部署（推荐）` - 括号内容应移至正文
- `## 2.3 国内一键安装（快速版）` - 括号内容应移至正文
- `### Kimi（月之暗面）★ 推荐` - 括号和符号应移至正文

**规范要求**：
> 标题中不加括号、破折号等注释（需要时移至正文）

**修改建议**：
```markdown
修改前：## 2.1 Mac本地部署（推荐）
修改后：## 2.1 Mac本地部署

修改前：### Kimi（月之暗面）★ 推荐
修改后：### Kimi
```

#### 2. 四级标题格式不规范

**问题位置**：
- `#### 为什么推荐Mac` - 应该是词组，不是完整句子
- `#### 为什么推荐飞书（国内）` - 应该是词组，不是完整句子
- `#### 为什么推荐Telegram（国外）` - 应该是词组，不是完整句子

**规范要求**：
> 四级标题：小四号黑体，前空两格，词组，句末无标点

**修改建议**：
```markdown
修改前：#### 为什么推荐Mac
修改后：#### Mac推荐理由

修改前：#### 为什么推荐飞书（国内）
修改后：#### 飞书推荐理由

修改前：#### 为什么推荐Telegram（国外）
修改后：#### Telegram推荐理由
```

#### 3. 目录格式问题

**问题位置**：
- 目录中使用了 `【Mac】`、`【云端】` 等非规范符号

**规范要求**：
> 禁止使用 【xxx】、√、❌、✅ 等非排版符号

### 二、正文规范问题（4个）

#### 1. ⭐ 口语化表述（高频错误）

**问题位置**：
- "如果你拥有Mac电脑，那么恭喜你！" - 包含"你"、"恭喜你"
- "让我们一步步完成安装" - 包含"我们"
- "让我们回顾一下本章的核心内容" - 包含"我们"

**规范要求**：
> 书面语，禁止口语：我们、咱们、大家、来聊一聊、手把手、恭喜你、告诉你

**修改建议**：
```markdown
修改前：如果你拥有Mac电脑，那么恭喜你！Mac本地部署将为你带来最佳的使用体验。
修改后：Mac电脑用户可以选择本地部署方式，获得最佳的使用体验。

修改前：让我们一步步完成安装。
修改后：按照以下步骤完成安装。

修改前：让我们回顾一下本章的核心内容。
修改后：本章核心内容回顾如下。
```

#### 2. ⭐ 使用了非规范符号

**问题位置**：
- 大量使用 `【Mac】`、`【云端】`、`【国内】` 等标记
- 使用了 `√`、`×`、`✅`、`❌` 等符号
- 使用了 `> 提示：` 的 Blockquote 格式

**规范要求**：
> 禁止使用 【xxx】、√、❌、✅ 等非排版符号
> 不使用 > 提示：、> 【xxx】 等 Blockquote 格式

**修改建议**：
```markdown
修改前：- 【Mac】 **Mac（强烈推荐）**
修改后：- Mac（强烈推荐）

修改前：- √ 原生支持最完善
修改后：- 原生支持最完善

修改前：> 【Mac】 **最佳体验**：如果你有Mac电脑...
修改后：注意：Mac电脑用户可获得最佳体验...
```

#### 3. ⭐ 英文单词加了双引号

**问题位置**：
- 部分英文单词可能加了引号（需要全文检查）

**规范要求**：
> 文中英文单词不加双引号
> 错误：单击"Next"按钮
> 正确：单击 Next 按钮

#### 4. 段落开头未空两格

**问题位置**：
- Markdown 转 Word 后需要检查

**规范要求**：
> 每段开头：空两个汉字位置

### 三、插图规范问题（7个）

#### 1. ⭐ 图题位置错误（高频错误）

**问题位置**：
- 所有图片的图题都在图片上方

**规范要求**：
> 图题在图的**下方**（不是上方）

**当前格式**：
```markdown
![Mac终端打开方式 - 通过Spotlight搜索Terminal](图片链接)
```

**正确格式**：
```markdown
如图2-1所示，通过 Spotlight 搜索打开 Terminal。

![](图片链接)

图2-1　Mac终端打开方式
```

#### 2. 图未按章编号

**问题位置**：
- 所有图片都没有编号

**规范要求**：
> 图按章编号，格式为 图X-X

#### 3. 正文中未引用图片

**问题位置**：
- 正文中没有使用 "如图X-X所示" 的引用格式

**规范要求**：
> 正文引用格式：如图X-X所示

#### 4. ⭐ 图中英文未翻译

**问题位置**：
- 需要检查所有截图中的英文是否已翻译

**规范要求**：
> 图中英文需翻译为中文

#### 5. 图的分辨率未确认

**问题位置**：
- 需要确认所有截图分辨率 ≥ 1000 dpi

#### 6. 图题格式不规范

**问题位置**：
- 图题应该使用 `图X-X　标题文字` 格式（全角空格）

#### 7. 文字描述与图的位置关系

**问题位置**：
- 需要确保文字描述在上方，图在下方

**问题位置**：
- 正文中没有使用 "见表X-X" 的引用格式

**规范要求**：

#### 4. 表格各项最后有标点

**问题位置**：

### 五、代码规范问题（2个）

#### 1. ⭐ 代码未使用"代码清单"样式

**问题位置**：
- Markdown 中的代码块转 Word 后需要应用"代码清单"样式

**规范要求**：
> 代码使用了**代码清单**样式

#### 2. 代码引出方式不规范

**问题位置**：
- 部分代码块前没有 "代码如下：" 或 "命令如下："

**规范要求**：
> 代码用 代码如下： 引出，命令用 命令如下： 引出

**修改建议**：
```markdown
修改前：
```bash
openclaw --version
```

修改后：
命令如下：

```bash
openclaw --version
```
```
